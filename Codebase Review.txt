 COMPREHENSIVE CODEBASE REVIEW

  CRITICAL ISSUES (Fix Immediately)

  1. N+1 Query Anti-Pattern in app/api/flocks/route.js:64-100

  // For each flock, 3 separate DB queries are made
  const enriched = await Promise.all(flocks.map(async (flock) => {
    const latestWeight = await prisma.weightRecord.findFirst({...});  // N+1
    const recentMortality = await prisma.mortalityRecord.aggregate({...});  // N+1
    const recentEggs = await prisma.eggProduction.aggregate({...});  // N+1
  }));
  Impact: 50 flocks = 150+ DB queries. Will cause timeout at scale.

  Fix: Use Prisma include with groupBy or batch loading with findMany + client-side mapping.

  2. Race Condition in Invoice Numbering (app/api/finance/supplier-invoices/route.js:12-24 &
  sales-invoices)

  const last = await prisma.supplierInvoice.findFirst({...});  // READ
  const seq = last ? parseInt(...) + 1 : 1;                     // COMPUTE
  // ... later ...
  await prisma.supplierInvoice.create({...});                   // WRITE
  Impact: Concurrent requests can receive identical invoice numbers.

  Fix: Use database sequence, Prisma transactions with retry, or unique constraint with
  application-level fallback.
